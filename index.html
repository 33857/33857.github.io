MYSQL中的列类型

December 1, 2018 
创建数据表的时候，指定的列可以存储的数据类型。
CRETE TABLE book （bid 列类型）
1、数值类
     TINYINT 微整形，占1个字节    范围-128~127
     SMALLlNT 小整形，占2个字节     范围-32768~32767
     INT  整形， 占4个字节     范围-2147483648~2147483647
     BIGINT 大整形  占8个字节
     FLOAT 单精度浮点型，占4个字节， 范围3.4e38，范围比INT大得多，可能产生计算误差
     DOUBLE  双精度浮点型，占8个字节，范围比BIGINT大得多。
     DECIMAL（M,D）定点小数，不会产生计算误差，M代表总的有效位数，D代表小数点后有效位数
     BOOL    布尔型，只有两个结果TRUE/1、1/0 。
2、日期类型
     DATE     日期型  ‘2018-12-13’
     TIME      时间型    ‘14:22:30’
     DATETIME   日期时间型  ‘2018-12-13  14:12:30’
3、字符类型
     VARCHAR（M）  变长字符串
     CHAR（M）   定长字符串，可能产生空间浪费，
      操作熟读较快，M最大值为255；用于存储手机号码，身份证号等固定长度的字符
     TEXT 大型变长字符，最多可以存2~4G
